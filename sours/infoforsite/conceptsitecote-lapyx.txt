

# Концепция сайта cote-lapyx.com

**cote-lapyx.com** — это командный сайт на 3 человек с общей главной страницей, общим блогом и отдельными персональными страницами каждого участника.

Команда:

* **1 участник** — верстка сайтов
* **2 участник** — разработка на **Java, C#**
* **3 участник** — дизайн в **Figma, WordPress**

Сайт должен одновременно решать 4 задачи:

* презентовать команду как digital/IT-студию
* показывать услуги и кейсы
* вести общий и личные блоги
* иметь авторизацию для владельцев и подписчиков

---

# Что должно быть на сайте

## 1. Главная страница

Общая страница команды.

Блоки:

* Hero-блок с названием **Cote Lapyx**
* краткое позиционирование команды
* кнопки: **Наши услуги / Проекты / Блог / Связаться**
* блок **О нас**
* блок **Наша команда**
* блок **Услуги**
* блок **Избранные проекты**
* блок **Общий блог**
* блок **Соцсети и мессенджеры**
* footer с контактами, копирайтом, ссылками

## 2. Страница “О нас”

Отдельно о команде:

* кто вы
* чем занимаетесь
* ваши сильные стороны
* подход к работе
* стек технологий

## 3. Общий блог

Общий блог команды:

* список статей
* категории
* теги
* поиск
* страница отдельной статьи
* комментарии
* лайки/подписка

## 4. Персональные страницы участников

Для каждого участника отдельная страница:

### Структура личной страницы:

* фото / аватар
* имя и специализация
* краткое био
* навыки
* инструменты / технологии
* личное портфолио
* личные проекты
* личный блог
* кнопки связи и соцсети

### Участник 1

* специализация: верстка сайтов
* стек: HTML, CSS, JS, адаптив, UI

### Участник 2

* специализация: Java / C#
* стек: Java, Spring Boot, C#, backend, API

### Участник 3

* специализация: Figma / WordPress
* стек: UI/UX, Figma, WordPress, дизайн систем

## 5. Портфолио / Проекты

Отдельный раздел:

* список проектов
* фильтрация по участникам / технологиям / типу
* карточка проекта
* изображения
* описание
* стек
* ссылка на demo / GitHub при наличии

## 6. Авторизация

Два типа пользователей:

### Владельцы

* вход в админ-панель
* создание и редактирование:

  * блогов
  * проектов
  * профилей
  * комментариев
  * контента сайта

### Подписчики

* регистрация / вход
* подписка на обновления
* комментарии к статьям
* избранное
* профиль пользователя

# Лучшая архитектура проекта

Вы ранее упоминали вариант **Next.js + Java backend** — это действительно хороший вариант.

## Рекомендуемый стек

### Frontend

* **Next.js**
* TypeScript
* Tailwind CSS
* Shadcn/UI или аналогичная UI-библиотека
* SSR/SSG для SEO
* адаптивная верстка

### Backend

* **Java Spring Boot**
* REST API
* JWT авторизация
* Spring Security
* Swagger/OpenAPI

### База данных

* **PostgreSQL**

### Хранение файлов

* локально на сервере или S3-совместимое хранилище

### Деплой

* Ubuntu VPS
* Nginx
* Docker
* SSL через Let’s Encrypt

# Почему именно такая архитектура

Подходит для:

* красивого современного интерфейса

* быстрой загрузки

## Java Spring Boot

Подходит для:

* надежного backend
* авторизации
* API
* работы с БД
* масштабирования в будущем
